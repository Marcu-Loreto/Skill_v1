O Problema: Query ≠ Intenção

Por que RAG ingênuo falha

  • Usuário escreve query vaga: "me explica mais sobre isso"
  • Embedding da query não captura a intenção real
  • Vocabulário do usuário ≠ vocabulário dos documentos
  • Perguntas multi-hop não são decompostas
  • Contexto de conversa anterior é ignorado

Resultado

Chunks recuperados não respondem a pergunta real. O LLM alucina ou dá respostas genéricas por falta de contexto relevante.


Dados

Queries brutas geram retrieval com recall 40-60% menor que queries reescritas (DMQR-RAG, ArXiv 2024). A lacuna semântica entre query e documento é o principal gargalo.

Fonte: ArXiv — DMQR-RAG (2024); NVIDIA Developer Blog (2025); TutorialQ Advanced RAG (2025)

Captura de Intenção

  • Resolve pronomes e referências anafóricas
  • Incorpora histórico de conversa (co-referência)
  • Identifica tipo de query: factual, comparativa, exploratória
  • Detecta ambiguidade e solicita clarificação ou infere

Geração de Nova Query

  • LLM reescreve com vocabulário alinhado aos documentos
  • Pode gerar múltiplas queries (Multi-Query) para cobertura
  • Pode gerar resposta hipotética (HyDE) como proxy de busca
  • Pode decompor em sub-queries (Query Decomposition)
Fonte: RQ-RAG (ArXiv, 2024); Query Understanding-Centric RAG (EmergentMind, 2025)

Técnicas de Query Rewriting

🔄 Multi-Query

Gera N variações da mesma pergunta. Une resultados de todas. Aumenta recall em 35%+ para queries ambíguas.

DMQR-RAG, ArXiv 2024

📄 HyDE

LLM gera resposta hipotética (sem docs). Embedding da resposta é usado como query. Melhora precision em domínios técnicos.

Gao et al., 2022

🧩 Decomposition

Quebra perguntas complexas em sub-queries independentes. Cada uma recupera chunks específicos. Essencial para multi-hop.

ArXiv 2507.00355

🔙 Step-Back

Generaliza a pergunta para conceito mais amplo antes de buscar. Útil quando a query é muito específica e não tem match direto.

Google DeepMind, 2024

💬 History-Aware

Condensa histórico de conversa + query atual em query standalone. Resolve "isso", "aquele", "o anterior". Essencial em chatbots.

LangChain, 2025

⚡ Q2E / Q2D

Query-to-Expansion e Query-to-Document. Técnicas da NVIDIA que adicionam termos relevantes ou geram pseudo-documentos para busca.

NVIDIA Blog, 2025

Implementação Prática

Pipeline recomendado

  • 1. Meça o baseline (RAGAS: recall, precision, faithfulness)
  • 2. Adicione History-Aware Rewriting (resolve 60% dos problemas)
  • 3. Adicione Multi-Query para queries ambíguas
  • 4. Avalie HyDE para domínios técnicos específicos
  • 5. Monitore custo vs. qualidade em produção

Custo

~$0.001-0.003 por query reescrita (gpt-4o-mini). ROI: menos alucinações = menos retrabalho = mais confiança.

# LangChain — History-Aware Retriever
from langchain.chains import create_history_aware_retriever

retriever = create_history_aware_retriever(
  llm=ChatOpenAI(model="gpt-4o-mini"),
  retriever=vectorstore.as_retriever(),
  prompt=rewrite_prompt
)

# MultiQueryRetriever
from langchain.retrievers import MultiQueryRetriever

mq_retriever = MultiQueryRetriever.from_llm(
  retriever=vectorstore.as_retriever(),
  llm=ChatOpenAI(model="gpt-4o-mini")
)
